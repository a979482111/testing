# 🚀 快速部署到服务器

## 方式 1：使用 Git 仓库（推荐）

### 步骤 1：创建远程仓库

选择任意 Git 平台创建新仓库：
- **GitHub**: https://github.com/new
- **Gitee (码云)**: https://gitee.com/new
- **GitLab**: https://gitlab.com/-/new

### 步骤 2：推送代码到远程仓库

**GitHub:**
```bash
# 替换为你的 GitHub 用户名和仓库名
git remote add origin https://github.com/你的用户名/snake-game.git
git branch -M main
git push -u origin main
```

**Gitee:**
```bash
# 替换为你的 Gitee 用户名和仓库名
git remote add origin https://gitee.com/你的用户名/snake-game.git
git branch -M main
git push -u origin main
```

### 步骤 3：在服务器上克隆并部署

```bash
# SSH 登录服务器
ssh root@你的服务器 IP

# 安装 Docker（首次需要）
curl -fsSL https://get.docker.com | sh

# 克隆代码
git clone <你的仓库地址>
cd snake-game  # 或你的仓库名

# 一键启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f
```

**访问：** `http://服务器 IP:3001`

## 方式 2：直接上传代码

如果不想用 Git，可以直接上传文件到服务器：

### 需要上传的文件：
```
backend/          # 后端代码（不含 node_modules 和 data）
src/              # 前端源码
dist/             # 前端构建产物（可选，服务器会重新构建）
package.json
pnpm-lock.yaml
vite.config.js
index.html
Dockerfile
docker-compose.yml
```

### 不需要上传的文件：
```
node_modules/     # 服务器会重新安装
dist/             # 可以重新构建
backend/data/     # 排行榜数据，服务器会创建
*.log
.env              # 敏感配置
```

## 常见问题

### Q: 服务器拉取代码太慢？
**A:** 使用 Gitee（码云）国内访问更快，或在 GitHub 使用镜像加速。

### Q: 私有仓库如何部署？
**A:** 
- GitHub: 使用 SSH 密钥或 Personal Access Token
- Gitee: 使用 SSH 密钥

生成 SSH 密钥：
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
cat ~/.ssh/id_ed25519.pub
# 复制输出内容到 Git 平台的 SSH 密钥设置
```

### Q: 如何更新代码？
```bash
# 在服务器上
cd /path/to/snake-game
git pull
docker-compose up -d --build
```
